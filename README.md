# EnergyMap: Платформа для расчёта электрических режимов

## 1. Техническое задание (Черновик)

### 1.1 Цель проекта
Цель проекта — создание цифровой платформы для расчёта режимов электрических сетей. 
В рамках текущего этапа работ рассматривается разработка фронтенд-части приложения, а также настройка взаимодействия фронтенда с сервером расчётов через API. Реализация вычислительного ядра сервера расчётов в данный блок задач не входит.

Платформа должна позволять:
- Строить и хранить схемы электрических сетей по территориям (деревни, ТП и т.д.) в **CAD-подобном интерфейсе**;
- Формировать **несколько сценариев** расчёта для одной схемы;
- Передавать топологию электрической сети **на сервер** для выполнения расчетов и отображать результаты **на клиенте**.



### 1.2 Пользователи и роли
* **Администратор**: управление пользователями, ролями, справочниками, шаблонами сценариев.
* **Инженер**: работа в каталоге, создание папок, схем, сценариев, запуск расчётов, формирование отчётов.
* **Пользователь**: просмотр схем/результатов/отчётов без прав редактирования.

### 1.3 Общая концепция интерфейса
* **Главная**: обзор и последние действия.
* **Каталог**: иерархическая структура (папки/объекты/схемы) — основное рабочее пространство.
* **Отчеты**: результаты выполненных расчетов.
* **Администрирование**: управление системой (доступно только Admin).

### 1.4 Редактор схемы (CAD-подобный)
Редактор должен поддерживать:
* Масштабирование, сетку и привязки.
* Слои отображения (геометрия/подписи/результаты).
* **Библиотеку элементов**: Источники/ТП, линии/участки, узлы/опоры, ответвления, потребители.

### 1.5 Сценарии расчёта
Пользователь может создавать несколько сценариев для одной версии схемы, настраивая:
* Сезонность (зима/лето).
* Типы нагрузок (типовые, индивидуальные, импорт из Excel).
* Панель «Готовность к расчёту» для проверки связности топологии и заполненности параметров.

### 1.6 Критерии приёмки
1. Авторизация под ролью Engineer с доступом к Каталогу.
2. Создание вложенной структуры папок и сущности «Схема».
3. Отрисовка схемы в редакторе с заполнением параметров элементов.
4. Создание, клонирование и модификация сценариев.
5. Сохранение версий схемы и формирование отчетов.
6. Администрирование пользователей и аудит действий.

## 2. Инструкция по запуску (Quick Start)

Проект для мониторинга и визуализации данных энергетики (в разработке).

### Текущая версия стека
- **Django**: 6.0.2  
- **MySQL**: 8.4  
- **Docker**: + Docker Compose

### Требования
- Установлен **Docker Desktop** → [docker.com](https://www.docker.com/products/docker-desktop/)
- Git

### Запуск проекта

1. **Склонируйте репозиторий:**
   ```bash
   git clone [https://github.com/stasevboris/energymap.git](https://github.com/stasevboris/energymap.git)
   cd energymap

```

2. **Соберите и запустите контейнеры:**
```bash
docker-compose up --build

```


* Первый запуск может занять 5–10 минут.
* После запуска откройте: [http://localhost:8000](https://www.google.com/search?q=http://localhost:8000)


3. **Остановить проект:**
`Ctrl + C` или `docker-compose down`.

### Полезные команды внутри Docker

* **Миграции БД:**
```bash
docker-compose exec web python manage.py migrate

```


* **Создать админа:**
```bash
docker-compose exec web python manage.py createsuperuser

```


* **Bash-терминал контейнера:**
```bash
docker-compose exec web bash

```



### Настройка стиля кода (обязательно)

Проект использует **pre-commit** (black + isort). Выполните один раз:

```bash
pip install pre-commit
pre-commit install

```
